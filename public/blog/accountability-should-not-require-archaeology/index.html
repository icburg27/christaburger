Accountability Should Not Require Archaeology

That is not a process. That is a scavenger hunt with legal exposure.

✦ Part of the series Things I've Learned Experimenting With AI

If accountability requires archaeology, the system is under-designed.

People should not need to dig through inboxes, Slack threads, meeting notes, and the memory of whoever was least distracted at the time.

That is not a process.

That is a scavenger hunt with legal exposure.

AI should make this better.

Every meaningful AI-assisted workflow should be able to answer:

  • What did it do?
  • What inputs did it use?
  • What changed?
  • What did it recommend?
  • What did a human approve?
  • Where is the evidence?

This is not just a security or compliance concern.

It is basic operational sanity.

Traceability is not paperwork. It is how trust survives scale.

(Building this into AI adoption is a core part of my governance practice.)
